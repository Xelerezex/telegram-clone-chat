## 1. Размеры строк

| **Таблица**         | **Расчет среднего размера строки**                                                                                                                        |         **Итог** |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------: |
| **storage_files**   | `16(id) + 8(file_type avg) + 80(object_key avg) + 8(size_bytes) + 8(created_at) = 120`                                                                    |            128 Б |
| **users**           | `16(id) + 24(username avg) + 16(phone) + 32(display_name avg) + 16(avatar_file_id) + 120(bio avg) + 8(created_at) + 8(updated_at) = 240`                  |            256 Б |
| **user_sessions**   | `64(token avg) + 16(user_id) + 24(device_id avg) + 96(user_agent avg) + 16(ip) + 8(created_at) + 8(expires_at) = 232`                                     |            256 Б |
| **messages**        | `16(id) + 16(chat_id) + 16(sender_id) + 8(seq_no) + 544(body из MMC=136×4) + 1(is_edited) + 1(is_encrypted) + 8(created_at) + 8(updated_at) = 618`        |            640 Б |
| **presence_cache**  | `16(user_id) + 24(device_id avg) + 8(status avg) + 8(last_seen_at) + 8(expires_at) = 64`                                                                  |             64 Б |
| **chats**           | `16(id) + 8(type avg) + 40(title avg) + 16(owner_id) + 16(chat_photo_file_id) + 16(last_message_id) + 1(is_secret) + 8(created_at) + 8(updated_at) = 129` |            160 Б |
| **chat_members**    | `16(chat_id) + 16(user_id) + 8(role avg) + 8(last_read_seq_no) + 4(unread_count) + 8(joined_at) + 8(updated_at) = 68`                                     |             80 Б |
| **update_buffer**   | `16(id) + 16(user_id) + 16(chat_id) + 16(message_id) + 16(update_type avg) + 64(payload avg) + 8(created_at) + 8(expires_at) = 160`                       |            160 Б |

| **файловый объект** | средний объект для MVP: аватар / фото чата                                                                                                                | 64 КБ = 65,536 Б |

## 2. Расчет QPS
| **Обозначение** | **Формула**                        | **Значение avg** | **Значение peak** |
| --------------- | ---------------------------------- | ---------------: | ----------------: |
| **Q_send**      | `DMS / 86,400`                     |         9,290.79 |         37,163.15 |
| **Q_fanout**    | `DMR / 86,400`                     |        59,722.22 |        238,888.89 |
| **Q_sync**      | `DMR / (Batch * 86,400)`           |         5,972.22 |         23,888.89 |
| **Q_session**   | `(DAU * DSU) / 86,400`             |         4,180.56 |         16,722.22 |
| **Q_event**     | `(DMS + DMR + DAU * DSU) / 86,400` |        73,193.56 |        292,774.26 |

## 5. Источник метрик

| **Метрика**                                      | **Источник**                   | **Комментарий**                                               |
| ------------------------------------------------ | ------------------------------ | ------------------------------------------------------------- |
| **DAU, DMS, DMR, DSU, Batch, k_peak**            | Расчетная база из раздела 2.   | Это твои исходные продуктовые и технические метрики.          |
| **Q_send, Q_fanout, Q_sync, Q_session, Q_event** | Посчитано на основе раздела 2. | Выводятся напрямую из исходных метрик                         |
| **Размер `messages.body = 544 Б`**               | Посчитано на основе раздела 2. | `136 символов × 4 байта`                                      |
| **Все суточные потоки данных**                   | Посчитано на основе QPS.       | Формула `size × operations/day`.                              |
| **Средние размеры строк кроме body**             | Проектные инженерные оценки    | Основаны на составе полей и средних длинах строк.             |
| **`users.write = 1 QPS`**                        | Допущение.                     | профиль редактируется редко.                                  |
| **`storage_files write = 5 QPS`**                | Допущение.                     | Консервативная оценка для MVP без медиавложений в сообщениях. |
| **`файловые объекты read = 600 QPS`**            | Допущение.                     | Оценка на чтение аватаров и фото чатов                        |
| **`event_log read = 0`**                         | Ограничение модели.            | Online-чтение аналитики в раздел 5 не включалось.             |
